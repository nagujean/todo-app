# 수용 테스트: 팀 관리 기능

## 개요

본 문서는 SPEC-TEAM-001 (팀 관리 기능)의 수용 테스트 시나리오를 정의합니다.

---

## 테스트 시나리오 (Gherkin 형식)

### Scenario 1: 소유자가 팀 삭제

```gherkin
Feature: 팀 관리 기능

  Scenario: 소유자가 팀 삭제
    Given 사용자가 "개발팀" 팀의 소유자이다
    And "개발팀" 팀에 5개의 할 일이 있다
    And "개발팀" 팀에 3명의 멤버가 있다
    When 사용자가 팀 관리 메뉴를 연다
    And "팀 삭제" 옵션을 클릭한다
    Then 팀 삭제 확인 다이얼로그가 표시된다
    When 사용자가 "개발팀"을 입력한다
    And "삭제" 버튼을 클릭한다
    Then 팀이 삭제된다
    And 모든 팀 데이터가 삭제된다
    And 사용자는 개인 모드로 전환된다
```

### Scenario 2: 멤버가 팀 탈퇴

```gherkin
  Scenario: 멤버가 팀 탈퇴
    Given 사용자가 "개발팀" 팀의 일반 멤버이다
    And "개발팀" 팀에 5명의 멤버가 있다
    When 사용자가 팀 관리 메뉴를 연다
    Then "팀 삭제" 옵션이 표시되지 않는다
    And "팀 탈퇴" 옵션이 표시된다
    When 사용자가 "팀 탈퇴" 옵션을 클릭한다
    Then 팀 탈퇴 확인 다이얼로그가 표시된다
    When 사용자가 "탈퇴" 버튼을 클릭한다
    Then 사용자가 팀에서 제거된다
    And 사용자는 개인 모드로 전환된다
```

### Scenario 3: 소유자는 팀 탈퇴 불가

```gherkin
  Scenario: 소유자는 팀 탈퇴 불가
    Given 사용자가 "개발팀" 팀의 소유자이다
    When 사용자가 팀 관리 메뉴를 연다
    Then "팀 탈퇴" 옵션이 표시되지 않는다
    And "팀 삭제" 옵션만 표시된다
```

### Scenario 4: 팀 이름 불일치 시 삭제 불가

```gherkin
  Scenario: 팀 이름 불일치 시 삭제 불가
    Given 사용자가 "개발팀" 팀의 소유자이다
    When 사용자가 팀 삭제 다이얼로그를 연다
    And "다른팀"을 입력한다
    Then "삭제" 버튼이 비활성화되어 있다
    When 사용자가 "개발팀"을 입력한다
    Then "삭제" 버튼이 활성화된다
```
