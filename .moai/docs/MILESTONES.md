## 1. MVP Definition of Done (MVP 완료 기준)

MVP가 완료되기 위해 충족해야 할 모든 기준을 정의합니다.

### 1.1 기능적 완료 기준

**필수 기능 (Must Have):**
- [x] 할 일 추가 기능 (Story 1.1) - **완료**
  - 텍스트 입력 후 추가 버튼 또는 Enter 키로 할 일 생성
  - 빈 입력 검증 및 에러 메시지 표시
  - UUID 기반 고유 ID 생성
  - 최상단에 새 할 일 표시

- [x] 할 일 완료/미완료 토글 (Story 1.2) - **완료**
  - 체크박스 클릭으로 상태 변경
  - 완료 시간 기록 (completedAt)
  - 시각적 구분 (취소선, 투명도)

- [x] 할 일 삭제 기능 (Story 1.3) - **완료**
  - 단일 삭제 및 페이드아웃 애니메이션
  - 완료 항목 일괄 삭제 기능

- [x] 할 일 필터링 (Story 1.4) - **완료**
  - 전체/미완료/완료 필터 탭
  - 필터 상태 localStorage 영속화

- [x] 데이터 영속화 (Story 1.5) - **완료**
  - localStorage 자동 저장
  - 페이지 새로고침 후 데이터 복원
  - UTF-8 JSON 형식 저장

**반응형 레이아웃 (Story 2.2):**

## 2. 단계별 마일스톤 (Phase-based Milestones)

바이브 코딩 개발 속도를 고려하여 기간 없이 단계 기반으로 정의합니다.

### 2.1 1단계: MVP 완성

**목표:** 핵심 할 일 관리 기능 구현 및 사용 가능한 제품 출시

**포함 에픽:**
- Epic 1: 할 일 관리 (전체)
- Epic 2: 사용자 경험 (반응형 레이아웃)

**완료 기준:**
- 모든 P0 스토리 완료
- E2E 테스트 커버리지 80%+
- Lighthouse 성능 점수 90+
- WCAG 2.1 Level AA 준수
- 크로스 브라우저 테스트 통과

**성과 지표:**
- 사용자가 할 일 추가/완료/삭제 가능
- 데이터 영속화로 페이지 새로고침 후 데이터 보존
- 모바일/태블릿/데스크톱 반응형 지원

**다음 단계 진입 조건:**
- MVP DoD 모든 항목 충족
- 사용자 피드백 수집 완료
- 주요 버그 수정 완료

---

### 2.2 2단계: 기능 확장

**목표:** 사용자 경험 개선 및 데이터 관리 기능 추가

**포함 에픽:**
- Epic 2: 사용자 경험 (다크 모드, 애니메이션)
- Epic 3: 데이터 관리 (내보내기/가져오기)

**완료 기준:**
- 모든 P1 스토리 완료
- 다크 모드 정상 작동 (시스템 테마 동기화)
- 60fps 애니메이션 유지
- 데이터 내보내기/가져오기 기능 완료
- 스키마 검증 및 병합 전략 테스트 통과
- 대용량 데이터(1000개) 처리 테스트 통과

**성과 지표:**
- 사용자가 테마를 자유롭게 전환 가능
- 데이터 백업/복원으로 안정성 확보
- 부드러운 애니메이션으로 사용 경험 개선

**다음 단계 진입 조건:**
- 2단계 모든 기능 완료
- 사용자 피드백 반영
- 성능 저하 없음 확인

---

### 2.3 3단계: 플랫폼화

**목표:** 클라우드 동기화, 계정 시스템, 팀 협업 기능 도입

**포함 에픽 (예정):**
- Epic 4: 클라우드 동기화 (Firebase/Supabase)
- Epic 5: 계정 시스템 (이메일/소셜 로그인)
- Epic 6: 팀 협업 (공유 작업 목록)
- Epic 7: 고급 기능 (태그, due date, 알림)

**완료 기준:**
- 클라우드 동기화 안정화
- 실시간 협업 기능 작동
- 모바일 앱 출시 (React Native 또는 Capacitor)
- 머신러닝 기반 작업 추천 (프로토타입)

**성과 지표:**
- 월간 활성 사용자(MAU) 1,000명 달성
- 사용자 유지율(Day 30) 30% 이상
- 평균 작업 완료율 70% 이상

**비즈니스 목표:**
- 프리미엄 구독 모델 도입 준비
- API 공개 및 서드파티 생태계 조성

## 4. Story별 Definition of Done

각 사용자 스토리가 완료되기 위해 충족해야 할 기준을 정의합니다.

### 4.1 Story 1.1: 할 일 추가하기 (3pt)

**Story DoD:**
- [ ] 인수 조건 AC-001 충족
- [ ] 빈 제목 검증 및 에러 메시지
- [ ] Enter 키 지원
- [ ] UUID v4 ID 생성
- [ ] 최상단 추가 로직
- [ ] E2E 테스트 작성 및 통과
- [ ] 코드 리뷰 승인

**테스트 케이스:**
- 정상 추가: "과제 제출하기" 입력 후 추가
- 빈 입력: 공백만 입력 후 추가 (에러 메시지)
- Enter 키: 입력 후 Enter 클릭 (추가 버튼과 동일)
- 200자 초과: 201자 입력 (자동 잘림 또는 에러)

---

### 4.2 Story 1.2: 할 일 완료하기 (2pt)

**Story DoD:**
- [ ] 인수 조건 AC-002 충족
- [ ] 체크박스 클릭으로 상태 토글
- [ ] 완료 시간 기록 (completedAt)
- [ ] 시각적 구분 (취소선, 투명도)
- [ ] 완료 항목 하단 이동 (필터 모드 아닐 때)
- [ ] E2E 테스트 작성 및 통과
- [ ] 코드 리뷰 승인

**테스트 케이스:**
- 완료: 체크박스 클릭 (완료 상태, 취소선)
- 미완료로 복귀: 완료된 항목 체크박스 다시 클릭
- 완료 시간: completedAt 필드에 ISO 8601 형식 저장
- 자동 정렬: 완료 시 하단으로 이동

---

### 4.3 Story 1.3: 할 일 삭제하기 (3pt)

**Story DoD:**
- [ ] 인수 조건 AC-003 충족
- [ ] 삭제 확인 다이얼로그
- [ ] 페이드아웃 애니메이션 (300ms)
- [ ] Undo 기능 (5초간)
- [ ] localStorage에서 삭제
- [ ] E2E 테스트 작성 및 통과
- [ ] 코드 리뷰 승인

**테스트 케이스:**
- 삭제: 삭제 버튼 클릭 → 확인 → 항목 제거
- 취소: 삭제 버튼 클릭 → 취소 → 항목 유지
- Undo: 삭제 후 5초 이내 Undo 버튼 클릭 → 복원
- Undo 만료: 5초 경과 후 Undo 버튼 사라짐

---

### 4.4 Story 1.4: 할 일 필터링하기 (2pt)

**Story DoD:**
- [ ] 인수 조건 AC-005 충족
- [ ] 전체/미완료/완료 필터 탭
- [ ] 필터 상태 localStorage 영속화
- [ ] 페이지 새로고침 후 상태 유지
- [ ] 빈 결과 메시지 표시
- [ ] E2E 테스트 작성 및 통과
- [ ] 코드 리뷰 승인

**테스트 케이스:**
- 전체: 10개 할 일 모두 표시
- 미완료: 5개 완료, 5개 미완료 중 미완료만 5개 표시
- 완료: 완료된 5개만 표시
- 상태 유지: "미완료" 선택 후 새로고침 → "미완료" 유지
- 빈 결과: 할 일 없을 때 "표시할 할 일이 없습니다" 메시지

---

### 4.5 Story 1.5: 데이터 영속화 (3pt)

**Story DoD:**
- [ ] 인수 조건 AC-005 충족
- [ ] localStorage 키 `todo-app-data` 사용
- [ ] JSON 배열 형식 저장
- [ ] 100ms 디바운스 자동 저장
- [ ] 페이지 로드 시 데이터 복원
- [ ] UTF-8 인코딩
- [ ] E2E 테스트 작성 및 통과
- [ ] 코드 리뷰 승인

**테스트 케이스:**
- 저장: 3개 할 일 추가 → localStorage에 저장 확인
- 복원: 페이지 새로고침 → 3개 할 일 그대로 표시
- 상태 유지: 완료된 할 일 상태 유지
- 완료 시간: completedAt 정보 보존

---
